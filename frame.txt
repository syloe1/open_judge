下面对每个模块进行简单介绍：

HttpServer 模块：会监听网络中发送的 Http (Ajax) 请求，然后将请求数据封装到 JSON 变量中传给 Control 模块进行下一步处理，处理完之后会对结果状态码进行相应调整，并返回结果。

Control 模块：主要就是控制和调用后面除数据库的所有模块的接口，进行不同模块之间的业务逻辑处理，类似于上帝。并且会根据参数中的 Token 对请求进行权限检测。

AnnouncementList 模块：公告模块，封装所有来自 MongoDB 数据库关于公告的接口，并为 Control 模块提供接口。

CommentList 模块：评论模块，封装所有来自 MongoDB 数据库关于评论的接口，并为 Control 模块提供接口。

DiscussList 模块：讨论模块，封装所有来自 MongoDB 数据库关于讨论的接口，并为 Control 模块提供接口。

Judger 模块：判题机模块，主要为系统提供判题服务。

ProblemList 模块：题目模块，封装所有来自 MongoDB 数据库关于题目的接口，在此接口上添加了对每个题目测试用例文件的管理以及对题目内容查询进行的缓存处理，最后为 Control 模块提供接口。

SolutionList 模块：题解模块，封装所有来自 MongoDB 数据库关于题解的接口，并为 Control 模块提供接口。

StatusRecord 模块：状态模块，封装所有来自 MongoDB 数据库关于状态的接口，在此接口上添加了对每个测评记录内容查询的缓存处理，最后为 Control 模块提供接口。

Tag 模块：标签模块，保存题目的所有标签并提供接口。

UserList 模块：用户模块，封装所有来自 MongoDB 数据库关于用户的接口，并添加了对用户鉴权的接口，最后为 Control 模块提供接口。

MongoDataBase 模块：MongoDB 数据库模块，会连接 MongoDB 数据库进行增删改查，并提供接口。

RedisDataBase 模块：Redis 数据库模块，会连接 Redis 数据库提供接口。

下面对系统工作流程进行介绍：

    HttpServer 模块监听 8081 端口。
    用户向网络中发送对后端数据的请求并携带请求参数。
    HttpServer 模块监听到请求，将参数封装成 JSON 变量，并传递给 Control 模块。
    Control 模块会调用一个或多个模块进行逻辑处理，这些模块会调用 MongoDB 数据库或 Redis 数据库请求数据或操作数据。
    Control 模块调用完成后将结果返回给 HttpServer 模块。
    HttpServer 模块会根据返回结果的 Result 参数调整不同的状态码并将最终结果进行返回。
